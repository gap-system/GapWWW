Since 1992 user contributed 'share packages' have been linked to the GAP system. They are not part of the main GAP library of functions but are distributed with GAP under their authors' names. The responsibility for updates, explanations, bug fixes, etc. remains with the author. Some of these packages have been written in the GAP language, other in programming languages such as C. Look at the individual pages for more information.

A set of programs has been given the status of a 'share package' only if it was thought to represent a piece of work equivalent to a sizeable mathematical publication. Since 1996 a formal refereeing system equivalent to that of a mathematical journal has been set up for the acceptance of share packages. The members of the GAP Council formed the editorial board.

The older share packages therefore had not undergone a formal refereeing process while all later ones (identifyable by the editor given) will have been refereed.

From the list of share packages given in the sequel you can reach brief descriptions for each of them.
