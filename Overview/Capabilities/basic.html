GAP provides many ways of forming

  • {% include ref.html manual="Reference: Lists" text="lists" %} (mutable and immutable), which also covers {% include ref.html manual="Reference: Sorted Lists and Sets" text="sets" %}, {% include ref.html manual="Reference: Row Vectors" text="row vectors" %}, {% include ref.html manual="Reference: Matrices" text="matrices" %}, {% include ref.html manual="Reference: Strings and Characters" text="strings" %}, and {% include ref.html manual="Reference: Boolean Lists" text="Boolean lists" %},
  • {% include ref.html manual="Reference: Records" text="records" %},
  • {% include ref.html manual="Reference: Domains and their Elements" text="domains" %}, i. e., objects representing sets with an algebraic structure.

Notions and tools modeled in some analogy to elementary set theory help to handle these:

  • distinction of {% include ref.html manual="Reference: Objects and Elements" text="objects and elements" %},
  • {% include ref.html manual="Reference: Booleans" text="Booleans" %},
  • {% include ref.html manual="Reference: Orderings" text="orderings" %},
  • {% include ref.html manual="Reference: Mappings" text="mappings" %} (in particular mappings respecting structures which again can be elements of structures),
  • {% include ref.html manual="Reference: Relations" text="relations" %}.

GAP can compute with

  • arbitrary {% include ref.html manual="Reference: Integers" text="integers" %},
  • {% include ref.html manual="Reference: Rational Numbers" text="rational numbers" %},
  • {% include ref.html manual="Reference: Cyclotomic Numbers" text="cyclotomic numbers" %}, in particular {% include ref.html manual="Reference: Gaussians" text="Gaussian numbers" %},
  • elements of {% include ref.html manual="Reference: Finite Fields" text="finite fields" %} (see also the coding theory package GUAVA),
  • {% include ref.html manual="Reference: p-adic Numbers (preliminary)" text="p-adic numbers" %},
  • {% include ref.html manual="Reference: Polynomials and Rational Functions" text="polynomials" %}, including {% include ref.html manual="Reference: Multivariate Polynomials" text="multivariate polynomials" %} (GAP contains only a basic implementation of Buchberger's algorithm for computing {% include ref.html manual="Reference: Groebner Bases" text="Gr√∂bner bases" %}, for more serious computations the package singular provides an interface to the extensive capabilities of the system SINGULAR),
  • {% include ref.html manual="Reference: Rational Function Families" text="rational functions" %} as well as
  • various kinds of group elements, e. g. {% include ref.html manual="Reference: Permutations" text="permutations" %}, {% include ref.html manual="Reference: Matrices" text="matrices" %}, and {% include ref.html manual="Reference: Words" text="abstract words" %}.

One can work with many algebracic structures. In addition to those for which links to separate pages are provided in the overview given on the page Mathematical Capabilities, there are e. g.

  • {% include ref.html manual="Reference: Finite Fields" text="finite fields" %},
  • {% include ref.html manual="Reference: Residue Class Rings" text="residue class rings" %} of integers (see also the package ResClasses),
  • {% include ref.html manual="Reference: Abelian Number Fields" text="abelian number fields" %},
  • {% include ref.html manual="Reference: Integral matrices and lattices" text="lattices and integral matrices" %}.

Also there are

  • various {% include ref.html manual="Reference: Combinatorics" text="combinatorial functions" %},
  • functions for {% include ref.html manual="Reference: Number Theory" text="elementary number theory" %} as well as
  • functions for {% include ref.html manual="Reference: Prime Integers and Factorization" text="prime number factorization" %} (see also the package FactInt).
